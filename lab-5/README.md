# Отчет по лабораторной работе №5  
**Алгоритмы отсечения отрезков и многоугольников**  

---

## 1. Цель работы  
Закрепление теоретического материала и практическое освоение основных алгоритмов отсечения отрезков и многоугольников. Разработка веб-приложения, визуализирующего работу следующих алгоритмов:  
- **Часть 1**: Алгоритм Сазерленда–Коэна для отсечения отрезков прямоугольным окном.  
- **Часть 2**: Алгоритм Кируса–Бека для отсечения отрезков выпуклым многоугольником.

---

## 2. Теоретическая часть  

### 2.1 Алгоритм Сазерленда–Коэна  
Алгоритм используется для отсечения отрезков прямоугольным окном. Основная идея заключается в разбиении плоскости на 9 областей с помощью кодов (код региона). Каждому концу отрезка присваивается 4-битный код, определяющий его положение относительно окна. Отсечение выполняется путём последовательного обрезания отрезка по границам окна до тех пор, пока он не окажется полностью внутри или снаружи.

**Коды областей:**  
- `0000` — внутри окна;  
- `0001` — слева;  
- `0010` — справа;  
- `0100` — снизу;  
- `1000` — сверху.  

Отсечение выполняется путём вычисления пересечений отрезка с границами окна через явное уравнение прямой.

### 2.2 Алгоритм Кируса–Бека  
Алгоритм предназначен для отсечения отрезков произвольным выпуклым многоугольником. Он использует параметрическое представление отрезка и нормали к рёбрам многоугольника. Для каждого ребра вычисляется скалярное произведение, определяющее, в какой полуплоскости относительно ребра находится точка. На основе этих вычислений определяются параметры `t_in` и `t_out`, которые задают видимую часть отрезка.

**Основные шаги:**  
1. Для каждого ребра многоугольника вычисляется внутренняя нормаль.  
2. Определяется вектор направления отрезка.  
3. Вычисляются параметры входа и выхода отрезка относительно каждой грани.  
4. Если `t_in ≤ t_out`, отрезок имеет видимую часть.

## 3. Реализация  

### 3.1 Используемые технологии  
- **HTML/CSS** — структура и стилизация интерфейса;  
- **JavaScript** — логика алгоритмов и отрисовка на элементе `<canvas>`;  
- **Формат ввода** — текстовый файл или ручной ввод в текстовое поле.

### 3.2 Структура приложения  
Приложение состоит из:  
1. **Панели управления** с вкладками для выбора части работы, полем ввода данных и кнопками управления.  
2. **Холста** для визуализации координатной сетки, исходных данных, окна отсечения и результата.  
3. **Легенды** для обозначения цветов элементов.

### 3.3 Алгоритмическая часть  
- **Часть 1**: Реализован алгоритм Сазерленда–Коэна через явное уравнение прямой.  
- **Часть 2**: Реализован алгоритм Кируса–Бека для выпуклого многоугольника.  

Оба алгоритма работают в системе координат с началом в верхнем левом углу холста.

## 4. Результаты работы  

### 4.1 Интерфейс  
Приложение предоставляет интуитивно понятный интерфейс:  
- Возможность загрузки данных из файла или ввода вручную.  
- Переключение между двумя частями работы с соответствующими подсказками по формату.  
- Визуализация сетки с шагом 50 единиц для удобства ориентирования.

### 4.2 Визуализация  
- **Исходные отрезки** отображаются красным цветом с пунктиром.  
- **Окно отсечения** (прямоугольник или многоугольник) — синим контуром.  
- **Видимые части отрезков** — зелёным цветом с утолщённой линией.

### 4.3 Примеры работы  
- **Часть 1**: Отсечение отрезков прямоугольным окном (150,150)–(450,450).  
- **Часть 2**: Отсечение отрезков выпуклым четырёхугольником (ромбом).

## 5. Заключение  
В ходе работы были успешно реализованы и визуализированы два алгоритма отсечения:  
1. **Сазерленда–Коэна** для прямоугольного окна.  
2. **Кируса–Бека** для выпуклого многоугольника.  

Приложение корректно обрабатывает входные данные, отображает все этапы отсечения и позволяет наглядно оценить работу алгоритмов. Реализация соответствует варианту №4 и требованиям лабораторной работы.
