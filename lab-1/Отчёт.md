# Отчет по проекту: "Лабораторная работа #1"

## Введение

Цветовые модели являются фундаментальным понятием в компьютерной графике, дизайне и обработке изображений. Данный проект представляет собой интерактивное веб-приложение для изучения и визуализации преобразований между тремя цветовыми моделями: RGB, HSV и LAB.

**Актуальность**: Понимание принципов работы цветовых моделей необходимо для специалистов в области компьютерной графики, веб-дизайна, обработки изображений и компьютерного зрения.

## Теоретическая часть

### Цветовая модель RGB

**RGB** (Red, Green, Blue) — аддитивная цветовая модель, основанная на смешении трех основных цветов:

- **R** (Красный) - [0, 255]
- **G** (Зеленый) - [0, 255]  
- **B** (Синий) - [0, 255]

**Принцип работы**: Цвет формируется путем сложения световых потоков. Используется в мониторах, телевизорах и других устройствах отображения.

### Цветовая модель HSV

**HSV** (Hue, Saturation, Value) — интуитивная модель, основанная на восприятии цвета человеком:

- **H** (Оттенок) - [0°, 360°] - определяет тип цвета
- **S** (Насыщенность) - [0%, 100%] - интенсивность цвета
- **V** (Яркость) - [0%, 100%] - светлота цвета

**Преимущества**: Более удобна для подбора цветов человеком compared to RGB.

### Цветовая модель LAB

**LAB** (Lightness, a, b) — перцептивно-равномерная модель, разработанная для соответствия человеческому восприятию:

- **L** (Светлота) - [0, 100] - яркость цвета
- **a** - [-128, 127] - положение между зеленым и красным
- **b** - [-128, 127] - положение между синим и желтым

**Особенности**: Независима от устройства, используется в профессиональной полиграфии и обработке изображений.

### Математические преобразования

#### RGB → HSV
```javascript
function rgbToHsv(r, g, b) {
    r /= 255; g /= 255; b /= 255;
    const max = Math.max(r, g, b);
    const min = Math.min(r, g, b);
    const delta = max - min;
    
    let h = 0, s = 0, v = max * 100;
    
    if (delta !== 0) {
        s = (delta / max) * 100;
        if (max === r) h = ((g - b) / delta) % 6;
        else if (max === g) h = (b - r) / delta + 2;
        else h = (r - g) / delta + 4;
        h = Math.round(h * 60);
        if (h < 0) h += 360;
    }
    return { h: Math.round(h), s: Math.round(s), v: Math.round(v) };
}
```

#### RGB → LAB (через XYZ)
Преобразование выполняется в два этапа:
1. RGB → XYZ (с учетом гамма-коррекции)
2. XYZ → LAB (с использованием эталонного белого D65)

### Ключевые компоненты

#### 1. Пользовательский интерфейс

**Центральный цветовой пикер**:
- Интегрирован в превью цвета
- Позволяет выбирать цвет из стандартной палитры
- Автоматически обновляет все модели

**Три панели управления**:
- RGB: ползунки и поля ввода для R, G, B
- HSV: регулировка оттенка, насыщенности и яркости  
- LAB: настройка светлоты и цветовых координат

#### 2. Алгоритмическая часть

**Основные функции преобразования**:
- `rgbToHsv()` / `hsvToRgb()` - прямое и обратное преобразование
- `rgbToLab()` / `labToRgb()` - сложные преобразования через XYZ
- `rgbToHex()` / `hexToRgb()` - работа с HEX-форматом

**Особенности реализации LAB**:
```javascript
// Нормализация относительно стандартного освещения D65
x /= 0.95047;
z /= 1.08883;

// Нелинейное преобразование для перцептивной равномерности
x = (x > 0.008856) ? Math.pow(x, 1/3) : (7.787 * x) + 16/116;
```

#### 3. Механизм синхронизации

**Принцип работы**:
- Любое изменение в одной модели запускает пересчет двух других
- Центральный пикер является основным источником изменений
- Автоматическая валидация и коррекция значений

**Обработка ошибок**:
- Предупреждения при выходе за допустимые диапазоны
- Автоматическое обрезание некорректных значений
- Сохранение целостности данных между преобразованиями
